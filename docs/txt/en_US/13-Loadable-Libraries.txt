====================================================================================================
JxMake Loadable Libraries
====================================================================================================

As has been explained in the introduction chapter, JxMake loadable libraries can be loaded using:
    :::include  <LIBRARY_NAME>
    :::sinclude <LIBRARY_NAME>

There are three types of libraries:
    1. Simple utility libraries.
    2. Higher function libraries.
    3. Internal libraries that are only meant to be loaded by the two types of libraries above.

Some libraries read and use global and/or environment variables for configuration values. Some other
libraries have built-in targets to show help about their features.

The JxMake user data  directory is currently located at '$HOME/0-JxMake/data'.
The JxMake user tools directory is currently located at '$HOME/0-JxMake/tools'.

For build systems that support limiting the number of warning and/or error messages before the build
process terminates, the environment variable 'JXMAKELIB_MAX_WARNINGS_AND_ERRORS' is used. If this
environment variable is not defined, the default value is 5.

----------------------------------------------------------------------------------------------------

Please note that because for now the JxMake development process is more concentrated on adding and
improving features in the core build system ('jxm.jar'), these loadable libraries may not be updated
as frequently and thus may not support newer build tools (such as compilers & linkers), newer SDKs,
some MCU variants, etc.


----------------------------------------------------------------------------------------------------
List of User Loadable Libraries
----------------------------------------------------------------------------------------------------

Null
    An empty library. This library can be used for library loading tests.


BasicPlatformUtil
    A basic platform utility library. Despite this library can be loaded and used by users directly,
    it is still primarily intended to be included from the other libraries listed below.

    This library reads and uses the 'JXMAKELIB_DOWNLOAD_CHK_INTERVAL_SECONDS' environment variable.
    If the environment variable is not defined, the default value of 432000 seconds (5 days) will
    be used. This value is also used by all other libraries that perform program, tool, and utility
    downloads (please also read the description of the macros 'JXMAKELIB_DOWNLOAD_*' below).


GitHubTool
    GitHub REST API tools library.

    This library reads and uses the 'GITHUB_TOKEN' environment variable.


SourceForgeTool
    A library of SourceForge related tools.


OpenSSLUtil
    This library provides the means to find and check the availability of OpenSSL commands and their
    versions.


PythonUtil
    This library provides the means to find and check the availability of Python commands and their
    versions.

    This library reads and uses the 'JXMAKELIB_PYTHON2_MAX_VERSION' and
    'JXMAKELIB_PYTHON3_MAX_VERSION' global/environment variables.

DownloadIntelHex
    This library provides a means to download the Python IntelHex  library. All the downloaded files
    are stored in the JxMake user data directory.

    This library reads and uses the 'JXMAKELIB_DOWNLOAD_ONLY_IF_NOT_EXIST' global/environment
    variable if the '?onlyIfNotExist' function argument is not specified.

    This library reads and uses the 'JXMAKELIB_DOWNLOAD_INTELHEX_FORCE_TAG' global/environment
    variable if the '?dlExactTag' function argument is not specified.

    Cross-platform C/C++ build tool library for AVR. This library supports using AVR GCC from native

    On Windows, this library also reads and uses the 'ProgramFiles' environment variable.
